> 关联文件：
> - `.clinerules`：通用 Agent 工作准则（核心宪法 + 操作规范）

> - 新条目按时间倒序追加在「FAQ」段落顶部

> - 偶然事件可只记录，无需补规则；系统性问题才提炼到 `.clinerules`

## 四、五问复盘模板

```markdown
### YYYY-MM-DD: 简短标题

- **类型**：错误 / 用户纠正 / 用户建议 / 社区发现 / 架构决策
- **背景**：当时在做什么
- **问题**：出了什么问题

- **Q0（系统性 vs 偶然）**：这是单次失误还是会反复触发的模式？
- **Q1（错在哪）**：直接错误是什么
- **Q2（为什么错）**：思维盲点 / 维度遗漏 / 假设错误
- **Q3（现有规则为何没阻止）**：哪条规范该兜底但没兜住
- **Q4（规则调整）**：需要新增/修改/删除哪条规则
- **Q5（未来如何更早发现）**：下次什么信号能触发同类怀疑

- **关联规则**：影响到的 .clinerules 章节
- **来源**：谁提出的 / 从哪里学到的
```
